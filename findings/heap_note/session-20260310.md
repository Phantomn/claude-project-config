## NLM 조회 요약 (2026-03-12)
### Q1. 확정된 정보로 취약 패턴 후보 요약
- UAF: delete_note에서 free 후 notes[v] NULL 미처리
- UAF read/write: view_note/edit_note가 notes[v] 포인터만 검사
- seccomp 제약: ORW 중심(허용 syscall 범위 내에서만 동작)
반증 항목: seccomp 허용 목록은 기존 BPF 디코딩 결과에 의존하므로, 바이너리 재빌드 시 재검증 필요

### Q2. notes 배열/구조체 레이아웃 요약
- notes 배열: PIE 상대 0x4040, 16개 포인터
- note struct: 0x28 bytes (title[0x18], size QWORD, content ptr QWORD)
- create/view/edit/delete가 동일 레이아웃을 가정
반증 항목: 런타임 leak으로 notes 주소가 확인되지 않았으므로 PIE base 계산은 아직 검증 필요

### Q3. 추가 수집해야 할 증거 공백
- 원격 libc 동일성 검증(서비스 환경 libc 버전)
- 런타임 PIE base 산출을 위한 data 포인터 leak
- libc 내 `_r_debug` 대체 심볼/오프셋 확보
반증 항목: 로컬/원격 libc 불일치 시 기존 오프셋 기반 분석은 무효
