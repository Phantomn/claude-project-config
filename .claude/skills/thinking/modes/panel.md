# Panel 모드 (병렬 분석)

## 개요
문제 유형에 따라 **2-3개 전략을 자동 선택**하고, Task 도구로 **병렬** 실행한 뒤 결과를 합성한다.

## 전략 자동 선택 매트릭스

| 문제 유형 | 감지 키워드 | 전략 조합 |
|----------|------------|----------|
| 아키텍처 결정 | "아키텍처", "설계", "구조 결정" | ToT + Step-Back + Ulysses |
| 디버깅/장애 | "버그", "장애", "에러", "성능" | ReAct + CoT + OODA |
| 시스템 이해 | "어떻게", "동작 원리", "전체 흐름" | Self-Ask + CoT + Step-Back |
| 위험 평가 | "위험", "프로덕션", "마이그레이션" | Ulysses + OODA + Step-Back |
| 기본값 | (매칭 없음) | CoT + ToT + Step-Back |

## 실행 절차

### 1. 전략 조합 결정
사용자 입력의 키워드를 위 매트릭스와 매칭하여 전략 조합을 결정한다.

### 2. Task 도구 병렬 호출
각 전략의 `에이전트 프롬프트` 섹션을 읽어 Task 도구의 prompt로 전달한다.

```
Task(subagent_type="general-purpose",
  description="[전략명] 분석",
  prompt="[strategies/xxx.md의 에이전트 프롬프트]\n\n분석 대상: [사용자 질문]")
```

**모든 Task를 단일 메시지에서 병렬 호출**한다.

### 3. 결과 합성
메인 에이전트가 모든 Task 결과를 수신한 후 아래 템플릿으로 합성한다.

```markdown
## 다중 관점 분석 결과

### 관점별 요약
- **[전략1명]**: [핵심 결론 1-2문장]
- **[전략2명]**: [핵심 결론 1-2문장]
- **[전략3명]**: [핵심 결론 1-2문장]

### 수렴점 (합의)
[모든 관점이 동의하는 부분]

### 발산점 (불일치)
[관점 간 차이가 있는 부분과 각 근거]

### 종합 판단
[수렴/발산을 고려한 균형 잡힌 최종 결론]
```

## 주의사항
- Task의 `subagent_type`은 항상 `"general-purpose"` 사용
- 각 에이전트는 **독립적으로** Sequential Thinking MCP를 호출
- 합성은 **메인 에이전트**가 수행 (Hub-Spoke 패턴)
- 에이전트 프롬프트에 사용자 질문 원문을 반드시 포함
