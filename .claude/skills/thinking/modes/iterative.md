# Iterative 모드 (순차 에스컬레이션)

## 개요
**CoT(1차) → Step-Back(2차) → ReAct(3차)** 순서로 점진적으로 깊은 분석을 수행한다. 각 라운드 결과가 충분하면 **조기 종료**하고, 불충분하면 이전 결과를 컨텍스트로 전달하여 다음 라운드를 실행한다.

## 에스컬레이션 체인

| 라운드 | 전략 | 목적 | 조기 종료 조건 |
|--------|------|------|--------------|
| 1차 | CoT | 기본 분석 | 명확한 답이 나왔으면 종료 |
| 2차 | Step-Back | 추상화 + 근본 원인 | 1차에서 놓친 관점이 보완되면 종료 |
| 3차 | ReAct | 증거 기반 검증 | 항상 최종 (증거로 확인) |

## 실행 절차

### 라운드 1: CoT 분석
```
Task(subagent_type="general-purpose",
  description="CoT 1차 분석",
  prompt="[strategies/cot.md의 에이전트 프롬프트]

분석 대상: [사용자 질문]")
```

### 평가 1: 충분성 판단
메인 에이전트가 1차 결과를 검토:
- **충분**: 명확한 결론이 있고, 누락된 관점이 없다 → **종료**
- **불충분**: 표면적 분석에 그쳤거나, "왜"에 답하지 못했다 → **2차 진행**

### 라운드 2: Step-Back 분석
```
Task(subagent_type="general-purpose",
  description="Step-Back 2차 분석",
  prompt="[strategies/step-back.md의 에이전트 프롬프트]

이전 분석 결과 (CoT 1차):
---
[1차 결과 전문]
---

위 분석이 불충분했습니다. 한 단계 추상화하여 근본 원인을 파악하세요.

분석 대상: [사용자 질문]")
```

### 평가 2: 충분성 판단
- **충분**: 근본 원인이 식별되고 해결책이 논리적이다 → **종료**
- **불충분**: 가설 검증이 필요하거나, 실제 증거가 부족하다 → **3차 진행**

### 라운드 3: ReAct 분석 (최종)
```
Task(subagent_type="general-purpose",
  description="ReAct 3차 분석",
  prompt="[strategies/react.md의 에이전트 프롬프트]

이전 분석 결과:
---
[1차 CoT 결과]
---
[2차 Step-Back 결과]
---

위 분석들의 가설을 증거로 검증하세요. 실제 도구를 사용하여 확인하세요.

분석 대상: [사용자 질문]")
```

### 최종 합성
```markdown
## 반복 심화 분석 결과

### 분석 깊이: [N]라운드 (CoT → Step-Back → ReAct)

### 라운드별 핵심 발견
- **1차 (CoT)**: [핵심 1-2문장]
- **2차 (Step-Back)**: [1차 대비 추가 발견]
- **3차 (ReAct)**: [증거 기반 확인 결과]

### 최종 결론
[가장 깊은 라운드의 결론을 기반으로 종합]
```

## 주의사항
- 각 라운드는 **순차적** 실행 (이전 결과에 의존)
- 조기 종료 판단은 **메인 에이전트**가 수행
- 2차/3차 프롬프트에 **이전 결과 전문**을 반드시 포함
- 최대 3라운드로 제한 (무한 루프 방지)
